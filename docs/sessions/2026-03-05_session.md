### Phase 2: Root Cause Analysis

**Why samples were frozen:** The `update_sample_data.py` script had a broken regex in `update_metric_samples()`. The pattern `[^}}]` in an rf-string becomes `[^}]`, which cannot traverse nested `{...}` structures (like `levelDistribution: { ... }`) to reach the `sampleData` array. The regex silently failed to match, so samples were never replaced. The script ran in CI without errors but did nothing.

### Phase 3: Rewrite update_sample_data.py

Complete rewrite of the sample selection pipeline:

- **Fixed the regex** -- replaced with `find_sample_data_range()` using bracket-counting to reliably locate each metric's `sampleData` array boundaries
- **Added relevance filtering** -- per-metric keyword lists (`RELEVANCE_KEYWORDS` dict) ensure samples match their topic. The YouTube API returns a lot of noise from broad search terms (e.g., "Poppy Playtime" and "Bugatti" videos in Healthcare results). The keyword filter eliminates off-topic viral content.
- **Prioritized severity over engagement** -- selectors now pick L3 entries first, then L2, then L1, sorting by engagement within each level. Previously sorted purely by views, which surfaced the most viral but least relevant content.
- **Added emoji stripping** -- `strip_emoji()` using comprehensive unicode ranges
- **Added HTML entity decoding** -- `html.unescape()` converts `&#39;` -> `'`, `&amp;` -> `&`
- **Added cross-platform deduplication** -- removes duplicate titles within each metric
- **Added HN and CFPB sample sources** -- was only doing YouTube/Reddit/TikTok
- **Fixed `get_latest_file()`** -- sorts by filename (contains timestamps) instead of `mtime`, skips files with <5 rows (matches `update_metric_data.py` approach)
- **Skips TBD/placeholder URLs** in YouTube samples

## Key Decisions

1. **Relevance filtering at sample selection, not collection.** The underlying collector data has noise because YouTube API returns loosely related results. Fixing collectors is a bigger project. The noise averages out in scoring (hundreds of entries), but samples shown to users must be on-topic. Per-metric keyword lists in the sample selector are the right layer for this.
2. **Severity-first sample selection.** Sorting by L3 > L2 > L1 then by engagement ensures the most impactful, relevant stories surface. Pure engagement sorting picks viral but off-topic content.
3. **Removed specific numbers from dataSources.** Instead of updating stale figures (which would go stale again), converted them to source citations without specific values. CI updates the platform counts (e.g., "YouTube: 170 videos") but can't update Redfin median prices or BLS ratios.
4. **HTML entity decoding.** YouTube API returns HTML entities in titles. Decoding them at sample selection time keeps the TS file clean.
